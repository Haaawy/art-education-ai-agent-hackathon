# Demo Checklist - Art Education AI Agent

Use this checklist when recording the hackathon video.

## Platform Flow

- [ ] Open the art education platform home page.
- [ ] Sign in as a teacher or open an existing teacher session.
- [ ] Show the teacher dashboard.
- [ ] Show the Microsoft Copilot Agent card in the dashboard.
- [ ] Open a class or artwork list.
- [ ] Show a student artwork card.
- [ ] Show the "إجراءات الذكاء الاصطناعي" section.
- [ ] Demonstrate "اقتراح وصف للعمل".
- [ ] Click "تحليل العمل بالوكيل الذكي".
- [ ] Confirm the agent page opens with the selected artwork.
- [ ] Run the artwork analysis.
- [ ] Show the loading state.
- [ ] Show the structured analysis result cards.
- [ ] Highlight the analysis steps.
- [ ] Show performance level.
- [ ] Copy the student feedback.
- [ ] Copy the full report.

## Microsoft Copilot Flow

- [ ] Return to the agent page or teacher dashboard.
- [ ] Click the Microsoft Copilot Agent link.
- [ ] Confirm it opens in a new browser tab.
- [ ] Explain that access may require Microsoft 365 Copilot permissions inside the organization.
- [ ] Test a prompt inside Copilot, for example:

```text
حلل عمل طالب بعنوان "حديقتي الملونة" يستخدم ألوانًا زاهية وتكوينًا بسيطًا، واكتب تغذية راجعة مناسبة.
```

- [ ] Show Copilot's structured educational response.

## Privacy and Safety Flow

- [ ] Open `/privacy`.
- [ ] Point out the warning about avoiding sensitive student data.
- [ ] Open `/terms`.
- [ ] Point out that the agent is a helper and does not replace teacher judgment.

## Closing Shot

- [ ] Summarize the two-agent story:
  - In-platform agent for platform artwork data.
  - Microsoft Copilot Agent Builder version for Microsoft 365 usage.
- [ ] Mention preview fallback mode for reliable demos.
- [ ] Mention future API or Actions integration path.
